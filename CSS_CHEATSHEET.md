# SportArena: шпаргалка по CSS

## 1. Как подключаются стили

В `index.html` подключён один главный файл:

```html
<link rel="stylesheet" href="css/style.css">
```

Внутри `css/style.css` подключаются остальные файлы:

```css
@import "variables.css";
@import "header.css";
@import "main.css";
@import "footer.css";
```

Вывод:

- `style.css` - точка входа;
- `variables.css` - переменные;
- `header.css` - шапка и меню;
- `main.css` - hero, контент, карточки, новости, сайдбар, адаптив;
- `footer.css` - подвал.

## 2. Как устроены классы

Используется стиль, близкий к BEM:

- `block` - самостоятельный блок;
- `block__element` - элемент блока.

Примеры:

- `header`, `header__container`
- `hero`, `hero__title`
- `event-card`, `event-card__image`
- `hotel-card`, `hotel-card__link`

Плюсы:

- понятно, к чему относится стиль;
- меньше конфликтов между классами;
- легко сопровождать код.

## 3. Главные CSS-переменные

В `variables.css` переменные объявлены в `:root`.

Самые важные:

- `--primary` - основной чёрный фон;
- `--secondary` - тёмно-синий;
- `--accent` - красный акцент;
- `--background` - белый;
- `--text` - основной светлый текст;
- `--muted` - приглушённый белый;
- `--border` - цвет рамок;
- `--header-height` - высота шапки;
- `--transition` - общее время анимации.

Зачем нужны:

- не дублировать цвета;
- быстро менять тему;
- повторно использовать одинаковые значения.

## 4. Глобальная база

Главные глобальные правила:

```css
* {
    margin: 0;
    padding: 0;
    box-sizing: border-box;
}
```

Что это даёт:

- убираются стандартные отступы браузера;
- `box-sizing: border-box` делает размеры блоков предсказуемыми.

Ещё важное:

- `html, body` задают общий шрифт и чёрный фон;
- `h1, h2, h3` используют `Source Code Pro`;
- `a` наследуют цвет и без подчёркивания по умолчанию.

Внутри `sidebar`:

## 6. Header и навигация

### `.header`

Главное:

- `display: flex`;
- `justify-content: space-between`;
- `align-items: center`;
- `position: absolute`;
- `height: var(--header-height)`;
- `z-index: 10`.

Почему важно:

- шапка лежит поверх hero;
- из-за `position: absolute` hero компенсирует её через `margin-top`.

### Бургер-меню

Сделано без JavaScript:

- скрытый `input.nav__checkbox`;
- `label.nav__toggle`;
- список `.nav__list`.

Ключевой селектор:

```css
.nav__checkbox:checked ~ .nav__list
```

Как работает:

- checkbox отмечен -> меню раскрывается;
- checkbox не отмечен -> меню скрыто.

### Что могут спросить

Почему без JS работает меню?  
Потому что состояние хранит checkbox, а CSS через `:checked` меняет видимость списка.

## 7. Hero-блок

Состав:

- `.hero`
- `.hero__image`
- `.hero::after`
- `.hero__title`

### Главное по hero

- `.hero` имеет `position: relative`;
- картинка растягивается на весь блок;
- `object-fit: cover` не искажает изображение;
- `::after` создаёт затемняющий слой;
- заголовок центрируется через:

```css
top: 50%;
left: 50%;
transform: translate(-50%, -50%);
```

### Что сказать на защите

Текст читается поверх картинки, потому что используется затемняющий overlay через псевдоэлемент `::after`.

## 8. Основной layout

Контейнер:

```css
.layout-container {
    display: flex;
}
```

Что делает:

- строит двухколоночный макет;
- слева основной контент;
- справа сайдбар.

Связка:

- `.events-list { flex: 1; }` - занимает всё свободное место;
- `.sidebar { width: 300px; flex-shrink: 0; }` - фиксированная боковая колонка.

## 9. Карточки событий `event-card`

Что важно помнить:

- фон карточки задаётся через CSS;
- используется `background-image: url('../images/austris.jpg')`;
- есть затемнение через `background-color` и `background-blend-mode: overlay`;
- есть `box-shadow`, `border-radius`, hover-эффект.

Ключевые свойства:

- `transition: transform 0.3s`;
- `transform: scale(1.02)` на hover;
- `display: flex` у блока команд;
- `color: var(--muted)` у даты и описания.

Контентные изображения матчей идут уже из HTML:

- `champp.png`
- `nba.png`
- `tennis.png`

## 10. Секция новостей

### `.news`

Что важно:

- общий фон секции: `backroundddd.avif`;
- `position: relative`;
- `overflow: hidden`;
- `::before` создаёт тёмный слой.

Правило:

```css
.news > * {
    position: relative;
    z-index: 2;
}
```

Нужно, чтобы контент был выше overlay.

### `.news-item`

Карточка новости использует:

- `display: flex`;
- `flex-wrap: wrap`;
- `backdrop-filter: blur(10px)`;
- полупрозрачный фон;
- hover-увеличение.

### Важный момент

Порядок элементов задаётся через `order`:

- заголовок - `order: 1`
- текст - `order: 2`
- картинка - `order: 3`

### Обрезка текста

У заголовка:

- `white-space: nowrap`
- `overflow: hidden`
- `text-overflow: ellipsis`

У текста новости:

- `-webkit-line-clamp: 2`

Это ограничивает текст двумя строками.

## 13. Адаптивность

Основные диапазоны:

- до `425px`
- `426px-767px`
- `768px-1024px`
- `1024px+`

### Что меняется на маленьких экранах

- layout становится вертикальным;
- появляется бургер-меню;
- hero уменьшается по высоте;
- шрифты уменьшаются;
- sidebar становится на всю ширину.

### Что важно запомнить

На мобильных:

- не две колонки, а одна;
- навигация превращается в выпадающее меню;
- изображения и отступы уменьшаются.

## 14. Самые важные свойства, которые могут спросить

### `box-sizing: border-box`

Ширина и высота учитывают `padding` и `border`.

### `position: relative`

Создаёт систему координат для абсолютно позиционированных детей.

### `position: absolute`

Вынимает элемент из потока и позволяет точно позиционировать.

### `z-index`

Управляет слоями: кто сверху, кто снизу.

### `object-fit: cover`

Изображение заполняет блок без искажения.

### `flex: 1`

Элемент растягивается и занимает свободное место.

### `flex-shrink: 0`

Элемент запрещено сжимать.

### `transform`

Используется для:

- центрирования;
- масштабирования на hover;
- анимации меню.

### `transition`

Делает изменения плавными.

### `background-blend-mode: overlay`

Смешивает цветовую подложку и картинку фона.

### `backdrop-filter: blur(10px)`

Размывает фон позади элемента, создаёт стеклянный эффект.

## 17. Готовые короткие ответы

### Как подключаются стили?

В HTML подключён `style.css`, а внутри него через `@import` подтягиваются остальные CSS-файлы.

### Почему использовали `:root`?

Чтобы хранить общие цвета, размеры и анимации в одном месте.

### Как работает бургер-меню?

Через скрытый checkbox и селектор `.nav__checkbox:checked ~ .nav__list`, без JavaScript.

### Почему hero-заголовок по центру?

Потому что используется абсолютное позиционирование и `transform: translate(-50%, -50%)`.

### Почему текст читается поверх картинок?

Потому что используются затемняющие overlay-слои и смешивание фона.

### Как сделан адаптив?

Через медиазапросы: на мобильных layout становится вертикальным и включается бургер-меню.

## 18. Что запомнить в первую очередь

Если совсем мало времени, запомнить нужно это:

1. Стили подключаются через `style.css` и `@import`.
2. В проекте есть CSS-переменные в `:root`.
3. Структура классов близка к BEM.
4. Основной layout построен на Flexbox.
5. Шапка абсолютная, hero компенсирует её высоту.
6. Бургер-меню работает через checkbox hack.
7. Hero и news используют overlay через псевдоэлементы.
8. Карточки используют hover, тени, скругления, blend-mode.
9. Адаптив сделан через media queries.
10. Новые изображения теперь разделены на фоновые и контентные.
