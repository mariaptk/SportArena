# SportArena CSS Summary

## 1. Что это за документ

Этот файл - подробная шпаргалка по CSS-части проекта `SportArena`.
Его задача - помочь быстро объяснить:

- как подключаются стили;
- как устроена структура CSS;
- какие блоки есть на странице;
- какие селекторы к каким элементам HTML относятся;
- зачем используется каждое важное свойство;
- как работает адаптивность;
- какие приёмы применены для доступности, композиции, позиционирования и визуальных эффектов.

Документ основан на реальном коде из:

- `index.html`
- `css/style.css`
- `css/variables.css`
- `css/header.css`
- `css/main.css`
- `css/footer.css`

## 2. Как CSS подключается к странице

В `index.html` в `<head>` подключается один главный файл стилей:

- `css/style.css` - строка 12 в `index.html`

Внутри `style.css` через `@import` подтягиваются остальные стили:

- `@import "variables.css";`
- `@import "header.css";`
- `@import "main.css";`
- `@import "footer.css";`

Логика подключения такая:

1. Открывается `index.html`.
2. Подключается `css/style.css`.
3. `style.css` импортирует остальные CSS-файлы.
4. Браузер собирает общий каскад стилей.

`style.css` выступает как единая точка входа, а остальные файлы разделены по зонам ответственности:

- `variables.css` - тема и переменные;
- `header.css` - шапка и навигация;
- `main.css` - основная часть страницы;
- `footer.css` - подвал.

## 3. Общая архитектура CSS

Проект использует подход, близкий к BEM:

- `block` - самостоятельный блок;
- `block__element` - элемент внутри блока.

Примеры:

- `header`
- `header__container`
- `nav__link`
- `event-card__image`

Почему это удобно:

- по имени класса сразу понятно, к чему он относится;
- стили меньше конфликтуют;
- проще сопровождать код;
- легче объяснять структуру.

Основные блоки страницы:

- `skip-link`
- `header`
- `nav`
- `hero`
- `layout-container`
- `events-list`
- `event-card`
- `news`
- `news-item`
- `sidebar`
- `hotel-card`
- `sidebar-nav`
- `footer`

## 4. Связь HTML и CSS

CSS в проекте почти полностью строится на классах.  
HTML-элемент получает `class`, а CSS выбирает его по этому классу.

Примеры:

- `<section class="hero">` связывается с `.hero`
- `<h1 class="hero__title">` связывается с `.hero__title`
- `<article class="event-card">` связывается с `.event-card`
- `<a class="hotel-card__link">` связывается с `.hotel-card__link`

Когда браузер видит:

```html
<h1 class="hero__title">WELCOME TO SPORTARENA</h1>
```

он находит правило:

```css
.hero__title { ... }
```

и применяет к нему:

- размер шрифта;
- цвет;
- позиционирование;
- выравнивание;
- `z-index`.

## 5. Файл `css/style.css`: глобальная база

Этот файл задаёт основу для всего проекта.

### 5.1. `@import`

Первые строки:

- подключают остальные CSS-файлы;
- делят стили по смысловым частям;
- делают структуру проекта читаемой.

### 5.2. Глобальный сброс

```css
* {
    margin: 0;
    padding: 0;
    box-sizing: border-box;
}
```

Что это делает:

- `margin: 0` убирает внешние отступы по умолчанию;
- `padding: 0` убирает внутренние отступы браузера;
- `box-sizing: border-box` делает расчёт размеров предсказуемым.

Почему `box-sizing: border-box` важно:

- ширина и высота начинают включать `padding` и `border`;
- проще контролировать размеры блоков;
- адаптивная вёрстка становится стабильнее.

### 5.3. Базовые стили для `html, body`

```css
html, body {
    font-family: Arial, Helvetica, sans-serif;
    line-height: 1.6;
    color: #000;
    background: #000;
}
```

Назначение:

- задать основной шрифт страницы;
- задать базовую высоту строки;
- задать общий чёрный фон.

### 5.4. Общие стили заголовков

```css
h1, h2, h3 {
    font-family: 'Source Code Pro', monospace;
}
```

То есть:

- основной текст использует Arial/Helvetica;
- заголовки получают `Source Code Pro`.

### 5.5. Общие стили ссылок

```css
a {
    color: inherit;
    text-decoration: none;
}
```

Это даёт:

- наследование цвета от родителя;
- отключение стандартного подчёркивания;
- возможность локально дооформлять ссылки в отдельных блоках.

## 6. Файл `css/variables.css`: тема и переменные

Здесь объявлены CSS-переменные внутри `:root`.

`:root` - корневой элемент документа.  
Переменные из `:root` доступны практически по всей странице.

Зачем нужны переменные:

- не дублировать одни и те же цвета;
- централизованно хранить тему;
- переиспользовать размеры и анимации.

Переменные проекта:

- `--primary: #000` - основной чёрный;
- `--secondary: #1e3a8a` - тёмно-синий;
- `--accent: #dc2626` - акцентный красный;
- `--success: #059669` - зелёный;
- `--warning: #d97706` - оранжевый;
- `--background: #fff` - белый;
- `--text: #fff` - основной светлый текст;
- `--muted: rgba(255, 255, 255, 0.7)` - приглушённый белый;
- `--gradient` - заранее объявленный градиент;
- `--card-bg` - тёмный фон карточек;
- `--border` - цвет границы;
- `--header-height: 65px` - высота шапки;
- `--transition: 0.3s ease` - общее значение анимации.

Как используется переменная:

```css
color: var(--background);
```

Это значит: взять значение `--background`, то есть белый цвет.

## 8. Доступность: `.visually-hidden` и `.skip-link`

### 8.1. `.visually-hidden`

Используется для текста `Menu` внутри бургер-кнопки.

Назначение:

- текст скрыт визуально;
- остаётся доступным для скринридеров.

Это важный accessibility-приём.

### 8.2. `.skip-link`

В начале `body` есть ссылка:

```html
<a href="#main-content" class="skip-link">Skip to main content</a>
```

Для чего она нужна:

- пользователь клавиатуры может сразу перейти к основному контенту;
- не нужно каждый раз проходить через навигацию.

Как работает:

- по умолчанию ссылка скрыта выше экрана: `top: -40px`;
- при `:focus` становится видимой: `top: 6px`;
- `z-index: 100` выводит её поверх интерфейса.

### 9.1. Блок `.header`


Свойства блока:

- `display: flex` - включает flex-контейнер;
- `justify-content: space-between` - разносит логотип и навигацию по краям;
- `align-items: center` - выравнивает по вертикали;

## 10. Навигация: `nav`, burger menu и checkbox hack

Навигация сделана без JavaScript.

### 10.1. Структура в HTML

Внутри `<nav class="nav">` есть:

- скрытый `input type="checkbox"` с классом `nav__checkbox`;
- `label` с классом `nav__toggle`;
- список ссылок `ul.nav__list`.

### 10.2. Зачем нужен checkbox

Он хранит состояние меню:

- не отмечен - меню скрыто;
- отмечен - меню раскрыто.

### 10.3. Почему `label` связан с `input`

У `input` есть `id="nav-toggle"`, а у `label` - `for="nav-toggle"`.  
При клике по `label` переключается checkbox.

### 10.4. `.nav`

Свойства:

- `display: flex`;
- `align-items: center`;
- `position: relative`.

`position: relative` нужен, чтобы мобильный список можно было позиционировать абсолютно относительно блока навигации.

### 10.5. `.nav__checkbox`

Скрывает сам переключатель:

- `opacity: 0`;
- `width: 0`;
- `height: 0`;
- `pointer-events: none`;
- `position: absolute`.

### 10.6. `.nav__toggle`

Это визуальная кнопка бургер-меню:

- по умолчанию `display: none`;
- на мобильных становится `inline-flex`;
- имеет `cursor: pointer`;
- прозрачный фон;
- плавный переход через `transition`.

#### `.nav__toggle:focus-visible`

Показывает обводку при фокусе с клавиатуры:

- `outline: 2px dashed var(--accent)`;
- `outline-offset: 2px`.

Это улучшает доступность.

### 10.7. `.nav__toggle-icon` и псевдоэлементы

Бургер-иконка строится из:

- `span.nav__toggle-icon`;
- `::before`;
- `::after`.

Это три полоски:

- центральная;
- верхняя;
- нижняя.

Используется `currentColor`, поэтому полоски автоматически берут текущий цвет текста родителя.

### 10.8. `.nav__list`

На десктопе:

- `display: flex`;
- элементы стоят в ряд;
- `gap: 19px`;
- `list-style: none`.

На мобильных:

- направление меняется на колонку;
- меню становится выпадающим;
- скрывается через:
  - `max-height: 0`;
  - `overflow: hidden`;
  - `opacity: 0`;
  - `transform: translateY(-8px)`.

### 10.9. Главное правило раскрытия меню

```css
.nav__checkbox:checked ~ .nav__list
```

Что оно означает:

- если checkbox отмечен;
- нужно выбрать следующий соседний список того же уровня.

Что происходит:

- `max-height` увеличивается;
- `opacity` становится `1`;
- `transform` убирается.

Так появляется выпадающее меню.

### 10.10. `.nav__link`

Ссылки навигации:

- используют моноширинный шрифт;
- белые по умолчанию;
- при `:hover` становятся красными.

## 11. Главный экран: блок `.hero`

HTML:

```html
<section class="hero">
    <img class="hero__image">
    <h1 class="hero__title">
```

### 11.1. `.hero`

Свойства:

- `position: relative` - создаёт систему координат;
- `height: 719px` - фиксированная высота;
- `margin-top: var(--header-height)` - компенсирует абсолютную шапку;
- `background: linear-gradient(...)` - резервный фон.

### Почему `position: relative` важно

Потому что к границам `.hero` привязываются:

- `.hero__image`;
- `.hero::after`;
- `.hero__title`.

### 11.2. `.hero__image`

Свойства:

- `position: absolute`;
- `width: 100%`;
- `height: 100%`;
- `object-fit: cover`;
- `top: 0; left: 0`.

Это даёт:

- изображение занимает весь блок;
- не искажает пропорции;
- при необходимости обрезается по краям.

### 11.3. `.hero::after`

Псевдоэлемент создаёт затемняющий слой над картинкой:

- `content: ""`;
- абсолютное позиционирование на всю площадь;
- `background: rgba(0, 0, 0, 0.7)`.

Назначение:

- повысить контраст текста;
- сделать hero выразительнее.

### 11.4. `.hero__title`

Свойства:

- `position: absolute`;
- `top: 50%`;
- `left: 50%`;
- `transform: translate(-50%, -50%)`;
- `font-family: 'Cinzel', serif`;
- `font-size: 90px`;
- `letter-spacing: -0.03em`;
- `color: var(--background)`;
- `text-shadow`;
- `z-index: 2`;
- `text-align: center`;
- `max-width: 90%`.

Как работает центрирование:

- сначала верхний левый угол ставится в центр блока;
- затем `translate(-50%, -50%)` сдвигает заголовок назад на половину его собственных размеров.

Слои внутри hero идут так:

1. фон `.hero`;
2. изображение `.hero__image`;
3. overlay `.hero::after`;
4. текст `.hero__title`.

## 17. Адаптивность: как страница меняется на разных экранах

Медиазапросы есть в `header.css` и `main.css`.

Основные диапазоны:

- до `425px` - маленькие смартфоны;
- `426px-767px` - широкие мобильные;
- `768px-1024px` - планшеты;
- `1024px+` - десктоп.

### Зачем нужны медиазапросы

Они позволяют менять стили в зависимости от ширины экрана:

- перестраивать колонки;
- уменьшать шрифты;
- менять размеры изображений;
- включать бургер-меню.

## 18. Адаптив шапки

### До `767px`

Что меняется:

- уменьшаются внутренние отступы шапки;
- `height: auto`;
- показывается бургер-кнопка;
- список навигации превращается в выпадающее меню;
- размер ссылок уменьшается.

### От `768px` до `1024px`

Что меняется:

- уменьшается padding шапки;
- ссылки становятся чуть меньше;
- навигация выравнивается вправо.

### До `425px`

Ещё сильнее уменьшаются отступы и корректируется положение кнопки.

## 19. Адаптив основного контента

### До `425px`

Hero:

- высота уменьшается до `400px`;
- верхний отступ становится `calc(var(--header-height) + 10px)`;
- заголовок уменьшается до `50px`.

Layout:

- `.layout-container` становится колонкой;
- padding уменьшается до `10px`.

Sidebar:

- ширина становится `100%`.

Карточки:

- уменьшаются отступы;
- изображения сильнее подстраиваются под ширину.

### От `426px` до `767px`

Что происходит:

- layout тоже становится колонкой;
- hero-title становится `60px`;
- sidebar растягивается на всю ширину;
- внутри sidebar включается `display: flex`.

Правило:

```css
.sidebar > * {
    flex: 1;
}
```

означает, что все прямые дочерние элементы боковой панели стараются делить пространство гибко.

### От `768px` до `1024px`

Что меняется:

- заголовок hero становится `70px`;
- контейнер переходит в колонку;
- сайдбар занимает всю ширину;
- картинки новостей могут быть шире.

### От `1024px`

Возвращается десктопный режим:

- `.layout-container` снова `row`;
- `.sidebar` снова `300px`.

## 20. Как строятся блоки и как всё связано между собой

Общая структура страницы:

3. `header`
4. `main`
5. `hero`
6. `layout-container`
7. внутри `layout-container`:
   - `events-list`
   - `sidebar`

9. внутри `sidebar`:
   - заголовки
   - карточки отелей
   - блок навигации

Как блоки связаны между собой:

- через HTML-вложенность;
- через классы;
- через наследование;
- через Flexbox;
- через абсолютное позиционирование;
- через CSS-переменные;
- через якорные ссылки.

Примеры таких связей:

- `header` абсолютный, а `hero` компенсирует его высоту через `margin-top`;
- `nav__checkbox` управляет `nav__list` через `:checked ~`;
- `news::before` создаёт overlay, а `news > *` поднимает контент поверх него;
- `layout-container` управляет раскладкой `events-list` и `sidebar`;
- переменные из `:root` используются по всему сайту;
- ссылки `href="#events"`, `href="#hotels"`, `href="#news"` привязаны к `id` соответствующих секций.

## 22. Важные свойства, которые могут спросить отдельно

### `box-sizing: border-box`

Позволяет учитывать `padding` и `border` внутри общей ширины и высоты элемента.

### `flex: 1`

Позволяет элементу занять доступное свободное пространство внутри flex-контейнера.

### `flex-shrink: 0`

Запрещает элементу сжиматься, когда места мало.

### `object-fit: cover`

Изображение заполняет контейнер без искажения, лишние части могут быть обрезаны.

### `position: relative`

Создаёт локальную систему координат для абсолютно позиционированных дочерних элементов.

### `position: absolute`

Позволяет точно расположить элемент через `top/right/bottom/left` и исключает его из обычного потока.

### `z-index`

Определяет, какой слой находится выше по оси Z.

### `overflow: hidden`

Скрывает всё, что выходит за границы блока.

### `transition`

Делает изменение свойств плавным.

### `transform: scale(...)`

Масштабирует элемент.

### `transform: translate(...)`

Сдвигает элемент относительно его текущего положения.

### `background-blend-mode`

Смешивает фоновые слои, например цвет и изображение.

### `backdrop-filter`

Применяет фильтр не к самому элементу, а к тому, что находится позади него.

- понятное разделение файлов;
- единая точка входа `style.css`;

## 27. Готовые формулировки для ответа преподавателю

### Если спросят: “Как подключаются стили?”

В HTML подключается один основной файл `style.css`, а уже внутри него через `@import` подключаются `variables.css`, `header.css`, `main.css` и `footer.css`. Это делает структуру проекта модульной.

### Если спросят: “Почему вынесли переменные в `:root`?”

Чтобы централизованно хранить цвета, размеры и анимации. Это уменьшает дублирование и упрощает изменение темы сайта.

### Если спросят: “Как работает бургер-меню?”

Меню сделано без JavaScript. Используется скрытый checkbox и связанный с ним label. Когда checkbox отмечен, срабатывает селектор `.nav__checkbox:checked ~ .nav__list`, и список становится видимым.

### Если спросят: “Как центрируется заголовок в hero?”

Заголовок позиционируется абсолютно: `top: 50%`, `left: 50%`, а затем `transform: translate(-50%, -50%)` смещает его на половину собственной ширины и высоты, за счёт чего получается точное центрирование.

### Если спросят: “Почему текст виден на фоне изображения?”

Потому что поверх изображений используются затемняющие слои через псевдоэлементы и полупрозрачные фоны, например `.hero::after` и `.news::before`, а также смешивание фона в карточках через `background-blend-mode`.

### Если спросят: “Как реализована адаптивность?”

Через медиазапросы для нескольких диапазонов ширины экрана. На мобильных layout становится вертикальным, появляется бургер-меню, уменьшаются размеры текста и изображений, а на широких экранах используется двухколоночная структура.

### Если спросят: “Какие современные CSS-приёмы использованы?”

CSS custom properties, Flexbox, псевдоэлементы, псевдоклассы, media queries, `object-fit`, `backdrop-filter`, `text-overflow: ellipsis`, `-webkit-line-clamp`, `transform`, `transition`.

## 28. Итог

CSS в проекте построен по понятной схеме:

1. `style.css` собирает все стили.
2. `variables.css` задаёт тему.
3. `header.css` отвечает за шапку и мобильную навигацию.
4. `main.css` оформляет hero, контент, карточки, новости, сайдбар и адаптив.
5. `footer.css` оформляет подвал.

Из этой схемы уже можно уверенно объяснить почти любой вопрос по CSS-коду проекта.
